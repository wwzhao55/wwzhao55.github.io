javascript系列问题

javascript是学习前端人员的重要语言之一，所有有必要对js理解的很透彻.在javascript中，万物皆对象，但是对象任然有区分。

一、JS原型和原型链

上面说到，js都是对象，但是有区分，分为两大类，即：普通对象（Object）和函数对象（Function），就一般而言，通过new Function的都是函数对象，其他归类为普通对象。

function f1(){
    //todo
}
var f2 = function(){
    //todo
};
var f3 = new Function('x','console.log(x)');

var o1 = {};
var o2 = new Object();
var o3 = new f1();

console.log(
    typeof f1,//function
    typeof f2,//function
    typeof f3,//function
    typeof o1,//object
    typeof o2,//object
    typeof o3 //object
);
>> function function function object object object

注解：f1属于函数的声明，f2实际上是一个匿名函数，匿名函数赋值给f2，属于函数表达式，f3不常见，但也是函数对象！

Function 是JS自带的对象，在函数创建的时候，JS自动通过new Function()方式来构建对象。Javascript中创建对象有两种方式：对象字面量和使用new表达式，分别为o1和o2的创建，重点我们来o3，用到了new操作符，其中 o3是f1的实例化对象，但是o3是普通对象,这就涉及到了原型和原型链。

在JS中，每当创建一个函数对象f1 时，该对象中都会内置一些属性，其中包括prototype和_proto_, prototype即原型对象，它记录着f1的一些属性和方法。var o3 = new f1() ,在这样的一个过程中，f1和o3会分别形成两条独立的原型链: o3原型链：obj->f1.prototype->Object.prototype->null,f1原型链： f1->Function.prototype->Object.prototype->null,其实prototype的主要作用就是继承。 通俗一点讲，prototype中定义的属性和方法都是留给自己的“后代”用的，因此，子类完全可以访问prototype中的属性和方法。

它在普通对象和函数对象中都存在， 它的作用就是保存父类的prototype对象，JS在通过new 表达式创建一个对象的时候，通常会把父类的prototype赋值给新对象的__proto__属性，这样，就形成了一代代传承...

二、new操作符

new 运算符是用来实例化一个类，从而在内存中分配一个实例对象.

var o3 = new f1();

从上面我们知道o3继承了f1中的部分属性，所以我们理解o3和f1为继承关系。判断一下o3和f1.

cat instanceof Animal; //true

结果证明：o3是f1的实例化对象，但是我们发现还有下面的现象：

1 var L = A.__proto__;
2 var R = B.prototype;
3 if(L === R)
4   return true;

在javascript中, 通过new可以产生原对象的一个实例对象，而这个实例对象继承了原对象的属性和方法。因此，new存在的意义在于它实现了javascript中的继承，而不仅仅是实例化了一个对象！

三、js作用域和作用域链

  • 作用域：变量在声明他的函数体以及这个函数体嵌套的任意函数体内都是有定义的。
 1 function scope(){
 2     var foo = "global";
 3     if(window.getComputedStyle){
 4         var a = "I'm if";
 5         console.log("if:"+foo); //if:global
 6     }
 7     while(1){
 8         var b = "I'm while";
 9         console.log("while:"+foo);//while:global
10         break;
11     }
12     !function (){
13         var c = "I'm function";
14         console.log("function:"+foo);//function:global
15     }();
16     console.log(
17          foo,//global
18          a, // I'm if
19          b, // I'm while
20          c  // c is not defined
21     );
22 }
23 scope();
24 console.log(foo);// undefined
  1. scope函数中定义的foo变量，除过自身可以访问以外，还可以在if语句、while语句和内嵌的匿名函数中访问。 因此，foo的作用域就是scope函数体。
  2. 在javascript中，if、while、for 等代码块不能形成独立的作用域。因此，javascript中没有块级作用域，只有函数作用域。

注：如果函数scope中foo变量前没有var，此时第24行输出为"global",foo就成为全局变量，为window对象。

  • 作用域链：一个函数体内潜逃了多层函数体，并在不同的函数体内它定义了同一变量，当其中一个好拿书访问这个变量是，便会形成一条作用域链

 1 foo = "window";
 2 function first(){
 3     var foo = "first";
 4     function second(){
 5        var foo = "second";
 6        console.log(foo);
 7     }
 8     function third(){
 9        console.log(foo);
10     }
11     second(); //second
12     third();  //first
13 }
14 first();

当执行second时，JS引擎会将second的作用域放置链表的头部，其次是first的作用域，最后是window对象，于是会形成如下作用域链： second->first->window, 此时，JS引擎沿着该作用域链查找变量foo, 查到的是"second",同级函数体之间不互相影响。
